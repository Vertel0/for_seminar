# legacy — намеренно противной god-class (семинар Л3)

`forecast_processor.py` (~1600 строк) — учебное легаси: **один класс делает всё** —
тянет прогнозы из источников, парсит (json/xml/csv), приводит единицы, валидирует,
грузит факт, считает ошибку, складывает в три хранилища и шлёт алерты. Нарушает SRP,
DIP, OCP. **Целиком читать не нужно — в этом и урок.**

## Запуск

```bash
pip3 install --user --break-system-packages pytest   # если pytest ещё нет
python3 forecast_processor.py     # прогон на фикстурах, печатает summary
python3 -m pytest                  # характеризующие тесты (ваша сетка безопасности)
```

## Что внутри

- `forecast_processor.py` — монстр. Точка, вокруг которой всё крутится — метод
  **`ForecastProcessor.process()`**. С него и начинайте.
- `fixtures/` — входные данные: `primary_forecast.json` (°C), `backup_forecast.xml`
  (°F/mph/inHg), `third_forecast.csv` (Кельвины/км·ч), `actual_weather.csv` (факт),
  `stations.csv` (белый список станций).
- `tests/test_characterization.py` — **характеризующие тесты**: фиксируют ТЕКУЩЕЕ
  поведение кода (golden values). Рефакторите под зелёным — поведение не уехало.

## Как подступиться (а не утонуть)

1. **Не читать весь файл.** Найдите `process()` — это шов, вокруг которого всё.
2. Запустите `pytest` — убедитесь, что зелено. Это страховка.
3. Вырежьте **один** шов (источник прогнозов) за интерфейс, добавьте 2-й источник —
   тесты остаются зелёными. Остальной монолит не трогайте.
